Адаптер - паттерн, который позволяет объектам какого-либо пакета несовместимого друг с другом, работать в одном каком-то ключе,
то есть работать с одним интерфейсом или в совместной работе какой-то.

То есть у нас есть проект где мы используем пакет, у которого есть какой-то интерфейс и, соответственно, мы работаем в нашей системе с 
несовместимым данным интерфейсом, но нам нужно обязательно использовать какой-то функционал и, соответственно, мы строим адаптер,
который позволяет взять этот функционал из этого пакета и работать в нашей системе. То есть суть данного паттерна заключается в этом. 

Или еще пример, допустим, у нас есть аналитический сервис, который мы внесли в нашу программу из какого-то другого пакета и данный
сервис работает в формате xml, то есть все документы которые передаются и принимаются в данном сервисе работают в xml формате, а у нас 
исторически сложилось так, что все форматы - это json объекты, и соответственно, нам нужно как-то транслировать наши json объекты в 
xml объекты, соответственно, используя только тот интерфейс, который мы импортировали из этого пакета, тем самым мы делаем адаптер, который
реализует интерфейс из этого пакета и уже реализуем конвертер из нашего json формата в xml и используем тот же самый интерфейс, то есть суть
заключается в этом.
